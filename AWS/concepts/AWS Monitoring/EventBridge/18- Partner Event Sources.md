# Partner Event Sources

Amazon EventBridge integrates with numerous Software-as-a-Service (SaaS) providers through **Partner Event Sources**. These integrations allow external applications to publish events directly into your AWS account.

This enables seamless event-driven workflows between AWS and third-party platforms.

------------------------------------------------------------------------

# What are Partner Event Sources?

Partner Event Sources are events generated by SaaS providers and delivered to Amazon EventBridge.

Instead of polling APIs for changes, applications receive events automatically.

------------------------------------------------------------------------

# Why Use Partner Event Sources?

Benefits include:

- Real-time integration
- No polling required
- Loose coupling
- Simplified architecture
- Automatic event routing

------------------------------------------------------------------------

------------------------------------------------------------------------

# Partner Event Bus

When you accept a Partner Event Source, EventBridge automatically creates a **Partner Event Bus**.

Rules can then route events from this bus.

# Real-World Example

A monitoring alert is generated in Datadog.

```text
Datadog

↓

Partner Event Bus

↓

EventBridge Rule

↓

Lambda

↓

SNS

↓

Operations Team
```

No custom integration code is required.

------------------------------------------------------------------------

# Best Practices

- Enable only required partner integrations.
- Filter events using Event Patterns.
- Monitor event delivery.
- Secure Event Buses using Resource Policies.
- Archive important business events.

# Key Takeaways

- Partner Event Sources connect SaaS applications with EventBridge.
- Events are delivered through a Partner Event Bus.
- EventBridge supports many popular SaaS providers.
- Enables real-time, event-driven integrations.
