# AWS X-Ray Daemon

The AWS X-Ray Daemon is a software application responsible for receiving trace data from the X-Ray SDK and securely sending it to the AWS X-Ray service.

The daemon acts as an intermediary between your application and AWS X-Ray.

------------------------------------------------------------------------

# What is the X-Ray Daemon?

The X-Ray Daemon collects trace data generated by the SDK.

Instead of every application communicating directly with AWS X-Ray, the SDK sends data to the daemon.

The daemon batches requests and uploads them efficiently.

------------------------------------------------------------------------

# Why Use the Daemon?

The daemon provides several benefits:

- Reduces network overhead
- Buffers trace data
- Batches requests
- Improves reliability
- Simplifies communication with AWS X-Ray

------------------------------------------------------------------------

# Daemon Architecture

```text
Application

↓

X-Ray SDK

↓

UDP Port 2000

↓

X-Ray Daemon

↓

HTTPS

↓

AWS X-Ray
```

------------------------------------------------------------------------

# Communication

The SDK sends trace data to the daemon using:

```text
UDP Port 2000
```

The daemon then communicates with AWS X-Ray over HTTPS.

------------------------------------------------------------------------

# Daemon Responsibilities

The daemon:

- Receives segments
- Buffers requests
- Compresses data
- Sends telemetry
- Uploads trace segments
- Handles retries

------------------------------------------------------------------------

# Where Does the Daemon Run?

The daemon can run on:

- Amazon EC2
- Amazon ECS
- AWS Elastic Beanstalk
- On-premises servers
- Virtual machines

For AWS Lambda, AWS manages the daemon automatically.

------------------------------------------------------------------------

------------------------------------------------------------------------

# Daemon Telemetry

The daemon periodically sends telemetry information such as:

- Segments received
- Segments sent
- Upload failures
- Buffer usage

This helps monitor daemon health.

------------------------------------------------------------------------

# Failure Handling

If AWS X-Ray is temporarily unavailable:

- The daemon buffers trace data.
- It retries uploads automatically.
- Temporary network failures are handled gracefully.

------------------------------------------------------------------------

# Real-World Example

A Java application running on Amazon EC2 generates trace data.

```text
Java Application

↓

X-Ray SDK

↓

X-Ray Daemon

↓

AWS X-Ray

↓

Service Map
```

The application remains unaware of communication details with AWS X-Ray.

------------------------------------------------------------------------

# Best Practices

- Keep the daemon updated.
- Monitor daemon logs.
- Ensure UDP port 2000 is accessible.
- Attach appropriate IAM roles.
- Monitor daemon health using CloudWatch.

------------------------------------------------------------------------

# Key Takeaways

- The X-Ray Daemon receives trace data from the SDK.
- It batches and uploads trace data to AWS X-Ray.
- Communication between SDK and daemon uses UDP.
- The daemon improves performance and reduces network overhead.
- AWS Lambda manages the daemon automatically.
