# What is Amazon CloudWatch?

Amazon CloudWatch is a **fully managed monitoring and observability
service** provided by AWS that collects, monitors, analyzes, and
visualizes operational data from AWS resources, on-premises servers,
applications, and custom workloads.

It enables organizations to monitor the health, availability, and
performance of their infrastructure and applications in near real time.

CloudWatch collects data in the form of **metrics**, **logs**,
**events**, and **traces**, allowing teams to quickly identify issues,
automate responses, and maintain highly available systems.

------------------------------------------------------------------------

# Why Do We Need CloudWatch?

As applications grow in size and complexity, manually monitoring every
server, database, or application becomes impossible.

CloudWatch provides centralized monitoring by helping you:

-   Monitor AWS resources
-   Collect application logs
-   Track performance metrics
-   Detect failures automatically
-   Receive notifications
-   Trigger automated recovery actions
-   Build operational dashboards
-   Troubleshoot production issues

------------------------------------------------------------------------

# Core Monitoring Capabilities

Amazon CloudWatch provides several monitoring capabilities:

-   **Metrics** -- Numerical performance data collected over time.
-   **Logs** -- Centralized storage and analysis of application and
    system logs.
-   **Alarms** -- Automatically notify or perform actions when
    thresholds are crossed.
-   **Dashboards** -- Visualize operational data using graphs and
    widgets.
-   **Events** -- Integrates with EventBridge for event-driven
    automation.
-   **Synthetics Canaries** -- Monitor APIs and websites by simulating
    user behavior.
-   **Insights** -- Analyze logs and identify trends using powerful
    query tools.

# What Can CloudWatch Monitor?

CloudWatch can monitor AWS services such as:

-   Amazon EC2
-   Amazon S3
-   AWS Lambda
-   Amazon RDS
-   Amazon DynamoDB
-   Amazon ECS
-   Amazon EKS
-   Elastic Load Balancer (ELB)
-   Amazon API Gateway
-   AWS Step Functions
-   Amazon CloudFront

It can also monitor on-premises servers, hybrid environments, custom
applications, containers, and virtual machines using the CloudWatch
Agent.

------------------------------------------------------------------------

# Types of Data Collected

## 1. Metrics

Metrics are time-ordered numerical data representing resource
performance.

**Examples**

-   CPU Utilization
-   Memory Usage (CloudWatch Agent)
-   Network In/Out
-   Disk Read Operations
-   Lambda Invocations

## 2. Logs

Logs contain detailed records generated by applications and AWS
services.

**Examples**

-   Application logs
-   Web server logs
-   Lambda logs
-   VPC Flow Logs
-   CloudTrail logs

## 3. Alarms

Alarms evaluate metrics continuously and perform actions when thresholds
are met.

Example:

``` text
CPU Utilization > 80%
```

Possible actions:

-   Send an SNS notification
-   Trigger Auto Scaling
-   Invoke a Lambda function
-   Recover an EC2 instance

## 4. Dashboards

Dashboards provide a centralized graphical view of infrastructure and
can display:

-   Metrics
-   Graphs
-   Alarm status
-   Text widgets
-   Multiple AWS Regions

# Real-World Example

For an e-commerce application, CloudWatch monitors EC2 CPU, Load
Balancer requests, database storage, Lambda errors, and application
logs.

If CPU utilization exceeds **80%** for **5 minutes**, CloudWatch can:

1.  Change the alarm state to **ALARM**
2.  Send an SNS notification
3.  Trigger an Auto Scaling policy
4.  Launch a new EC2 instance
5.  Restore application performance
