# AWS Service Metrics

AWS automatically publishes CloudWatch metrics for many AWS services. These metrics help you monitor the health, performance, and utilization of your AWS resources without installing additional software.

Each AWS service publishes its metrics to a predefined namespace.

---

# What are AWS Service Metrics?

AWS Service Metrics are built-in metrics generated by AWS services and sent to CloudWatch automatically.

Examples include:

- EC2 CPU Utilization
- Lambda Invocations
- RDS Database Connections
- ELB Request Count
- DynamoDB Throttled Requests

Most AWS service metrics are available at no additional configuration.

# Amazon EC2 Metrics

Frequently monitored metrics:

- CPUUtilization
- NetworkIn
- NetworkOut
- DiskReadBytes
- DiskWriteBytes
- StatusCheckFailed

**Note:** Memory utilization is **not** published by default. Install the CloudWatch Agent to collect memory, disk usage, and swap metrics.

---

# AWS Lambda Metrics

Important metrics include:

- Invocations
- Errors
- Duration
- Throttles
- ConcurrentExecutions
- IteratorAge (for stream processing)

These metrics help identify performance and scaling issues.

---

# Amazon RDS Metrics

Key database metrics:

- CPUUtilization
- DatabaseConnections
- FreeStorageSpace
- ReadIOPS
- WriteIOPS
- ReadLatency
- WriteLatency
- FreeableMemory

Monitor these metrics to maintain database performance.

---

# Amazon S3 Metrics

Examples:

- BucketSizeBytes
- NumberOfObjects
- AllRequests
- GetRequests
- PutRequests
- 4xxErrors
- 5xxErrors

Storage metrics are updated periodically, while request metrics must be enabled.

# Elastic Load Balancer Metrics

- HTTPCode_Target_4XX_Count
- HTTPCode_Target_5XX_Count

# Amazon DynamoDB Metrics

Useful metrics:

- ConsumedReadCapacityUnits
- ConsumedWriteCapacityUnits

- WriteThrottleEvents

These metrics help detect throttling and capacity issues.

# Amazon ECS Metrics

Container metrics include:

- CPUUtilization
- MemoryUtilization
- RunningTaskCount
- PendingTaskCount

Enable Container Insights for enhanced monitoring.

---

# Choosing the Right Metrics

Focus on metrics that directly affect application availability and user experience.

Examples:

- High CPU utilization
- Increasing error rates
- Rising latency
- Low storage space
- Unhealthy load balancer targets

Create CloudWatch Alarms for these critical metrics.

---

# Best Practices

- Monitor the default metrics for every production service.
- Install the CloudWatch Agent for operating system metrics.
- Use dashboards to combine metrics from multiple services.
- Configure alarms for critical thresholds.
- Review metrics regularly for capacity planning.

---

# Key Takeaways

- AWS automatically publishes metrics for most managed services.
- Each service uses its own CloudWatch namespace.
- Built-in metrics provide immediate operational visibility.
- Combine service metrics with custom metrics for complete monitoring.
- Use alarms and dashboards to proactively detect issues.
