# Metric Filters

Metric Filters allow Amazon CloudWatch Logs to convert specific log patterns into CloudWatch metrics. Once a metric is created, it can be visualized on dashboards or monitored using CloudWatch Alarms.

Metric Filters bridge the gap between **logs** and **metrics**, enabling automated monitoring based on application log data.

---

# What are Metric Filters?

A Metric Filter scans incoming log events in a Log Group for matching patterns.

When a pattern matches, CloudWatch increments or publishes a metric.

Example:

```text
Application Log

ERROR Database connection failed
```

Metric Filter:

```text
Pattern: ERROR
```

Result:

```text
Metric: ApplicationErrors = 1
```

---

# Why Use Metric Filters?

Metric Filters help you:

- Detect application errors
- Monitor failed login attempts
- Track HTTP 4xx and 5xx responses
- Count security events
- Create alarms from log data
- Monitor business events

---

# How Metric Filters Work

```text
Application
      |
      v
CloudWatch Logs
      |
      v
Metric Filter
      |
      v
CloudWatch Metric
      |
      v
Alarm / Dashboard / SNS
```

Whenever a log matches the filter pattern, the associated metric is updated.

---

# Filter Patterns

CloudWatch supports several filter pattern types.

## Simple Text Match

```text
ERROR
```

Matches any log containing the word **ERROR**.

---

## Multiple Keywords

```text
ERROR Exception Timeout
```

Matches logs containing the specified keywords.

---

## JSON Pattern

Example log:

```json
{
  "status": 500,
  "service": "payment"
}
```

Pattern:

```text
{ $.status = 500 }
```

---

## Space-Delimited Pattern

Example:

```text
127.0.0.1 GET /login 500
```

Pattern:

```text
[ip, method, path, status = 500]
```

Useful for web server access logs.

# Creating CloudWatch Alarms

Once a metric is generated, it can trigger an alarm.

Example workflow:

```text
Application Logs
       |
       v
Metric Filter
       |
       v
ApplicationErrors Metric
       |
       v
CloudWatch Alarm
       |
       v
Amazon SNS Notification
```

This enables proactive monitoring without modifying application code.

---

# Real-World Example

An e-commerce application writes the following log:

```text
ERROR Payment gateway timeout
```

A Metric Filter searches for the keyword **ERROR**.

Every occurrence increments the **PaymentErrors** metric.

If more than 20 errors occur within five minutes, CloudWatch triggers an alarm and sends an SNS notification to the operations team.

---

# Best Practices

- Create filters only for important events.
- Use structured JSON logs whenever possible.
- Keep filter patterns simple and easy to maintain.
- Create alarms for critical metrics generated by filters.
- Review and update filters as applications evolve.

---

# Key Takeaways

- Metric Filters convert log events into CloudWatch metrics.
- They enable dashboards and alarms based on log data.
- Filter patterns support text, JSON, and space-delimited logs.
- Metric Filters are ideal for monitoring errors, security events, and business activities without changing application code.
