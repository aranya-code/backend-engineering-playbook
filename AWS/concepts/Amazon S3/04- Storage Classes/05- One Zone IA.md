# Amazon S3 One Zone-IA

---

# Introduction

Not all data requires protection across multiple Availability Zones. Some workloads contain data that can be recreated if lost or already exists elsewhere. Replicating these objects across multiple Availability Zones increases storage costs without providing meaningful business value.

Amazon **S3 One Zone-Infrequent Access (S3 One Zone-IA)** is designed for infrequently accessed data that does not require Multi-AZ resilience. By storing objects in a single Availability Zone, AWS offers significantly lower storage costs while still providing immediate access.

---

# Why S3 One Zone-IA Exists

Imagine a media company that generates temporary video rendering outputs.

The original video files are stored safely in another bucket and can always be regenerated.

The rendered output is accessed occasionally for quality checks, but losing it would not be catastrophic.

Using Amazon S3 Standard-IA would provide Multi-AZ redundancy that this workload does not actually need.

S3 One Zone-IA offers a more cost-effective alternative.

> 💡 **Engineering Insight**
>
> One Zone-IA is intended for **re-creatable** or **secondary** data—not your only copy of critical business information.

---

# Characteristics

| Feature | Amazon S3 One Zone-IA |
|---------|------------------------|
| Access Pattern | Infrequent |
| Retrieval Time | Milliseconds |
| Availability | 99.5% |
| Durability | 99.999999999% (11 nines) |
| Availability Zones | Single |
| Retrieval Charges | Yes |
| Minimum Storage Duration | 30 Days |

Objects remain immediately accessible but are stored in only one Availability Zone.

---

# Architecture

```text
            Client
               │
               ▼
      Amazon S3 One Zone-IA
               │
               ▼
     Availability Zone A
```

Unlike S3 Standard or S3 Standard-IA, data is **not** replicated across multiple Availability Zones.

If the Availability Zone is permanently lost, objects stored exclusively in that zone may become unrecoverable.

---

# Cost Model

Compared with S3 Standard-IA:

Advantages:

- Lower storage cost

Trade-offs:

- Single Availability Zone storage
- Retrieval charges
- 30-day minimum storage duration
- Lower availability target

The reduced cost reflects the lower level of infrastructure redundancy.

---

# Typical Workloads

Amazon S3 One Zone-IA is suitable for:

- Secondary backup copies
- Re-creatable datasets
- Temporary processing outputs
- Development and test environments
- Cached media
- Analytics staging data

These workloads can tolerate the unlikely loss of a single Availability Zone.

> 🏗️ **Production Pattern**
>
> Store the primary copy of important business data in Multi-AZ storage classes. Use One Zone-IA only for secondary or reproducible data.

---

# When Not to Use One Zone-IA

Avoid One Zone-IA for:

- Customer documents
- Financial records
- Medical records
- Legal documents
- Production databases
- Compliance-sensitive information
- The only copy of business-critical data

These workloads require Multi-AZ resilience.

---

# Best Practices

- Keep another copy of important data.
- Use Lifecycle Rules for automatic transitions where appropriate.
- Monitor retrieval frequency.
- Evaluate business recovery objectives before choosing One Zone-IA.

---

# Common Mistakes

- Using One Zone-IA for irreplaceable production data.
- Ignoring Availability Zone failure scenarios.
- Assuming durability is the same as availability.
- Moving active workloads into One Zone-IA solely to reduce storage costs.

> ⚠️ **Common Mistake**
>
> Lower storage cost comes from storing data in a single Availability Zone. Do not confuse lower cost with equivalent fault tolerance.

---

# Real-World Example

A machine learning team stores intermediate feature-engineering datasets.

The original raw data is securely stored in Amazon S3 Standard.

If intermediate datasets are lost, they can be regenerated by rerunning the data pipeline.

Using Amazon S3 One Zone-IA significantly reduces storage costs without increasing business risk.

---

# Key Takeaways

Amazon S3 One Zone-IA is designed for infrequently accessed data that can be recreated or already exists elsewhere. It offers lower storage costs by storing data in a single Availability Zone while maintaining immediate retrieval.
