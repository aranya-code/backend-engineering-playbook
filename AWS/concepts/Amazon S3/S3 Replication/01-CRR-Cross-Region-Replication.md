# Cross Region Replication (CRR)

## Definition
CRR automatically replicates objects from a source bucket to a destination bucket in another AWS Region.

## Why Use CRR?
- Disaster Recovery
- Compliance Requirements
- Geographic Proximity
- Multi-Region Architectures

## Example
Mumbai --> Singapore
Mumbai --> Frankfurt

## Benefits
- Regional resilience
- Compliance support
- Faster access for global users

## Exam Tip
CRR = Different Regions
